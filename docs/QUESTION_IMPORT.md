# 画像付き問題の登録

## Macから一括登録する

問題JSONの各項目に、対応する画像のファイル名を`imageFile`で指定します。

```json
[
  {
    "subject": "歯内療法学",
    "subcategories": ["第1章", "歯髄診断"],
    "question": "画像の所見を答えてください。",
    "answers": ["不可逆性歯髄炎"],
    "explanation": "自発痛と温熱痛を確認します。",
    "orderedAnswers": false,
    "imageFile": "endo_001.png"
  }
]
```

1. 問題管理の一括登録でJSONを選択します。
2. 画像選択欄で、JSONから参照する画像をすべて選択します。iCloud Drive上の教科フォルダもファイル選択画面から開けます。
3. 「検証する」で対応漏れ、同名画像、重複問題、保存サイズを確認します。
4. 「検証OKデータを一括追加」で画像アップロードとFirestore保存を実行します。

`imageFile`は相対パスでの一致を先に試し、ブラウザがフォルダ情報を渡さない場合はファイル名で照合します。異なる教科に同名画像があると判定できないため、画像ファイル名は一括登録内で一意にしてください。

登録時は全画像をStorageへアップロードした後、問題と関連状態をFirestoreへ一括保存します。保存が失敗した場合は問題の追加を取り消し、追加途中のStorage画像も削除します。問題データはFirestoreの1ドキュメント上限に達しないよう自動的に複数文書へ分割されるため、JSONを手作業で分割する必要はありません。従来の1文書形式で保存された問題も引き続き読み込めます。

## iPhone/iPadから登録・更新する

問題管理で対象問題を選び、画像ファイルを選択して「更新」を押します。新しい問題は入力後に「追加」を押します。この経路は従来どおり1問題につき1画像を扱い、同じユーザーの`users/{uid}/questions/{questionId}/`配下へ保存します。

## Codexで教材画像を準備する場合

教科ごとのフォルダで画像を作成し、重複しない名前を付けます。JSONの`imageFile`にはそのファイル名を記載してください。macOSがCodexプロセスのiCloud Driveアクセスを拒否する場合でも、アプリ側のファイル選択画面から画像を選べます。
