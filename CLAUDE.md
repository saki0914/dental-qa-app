# CLAUDE.md

## 運用ルール

- 作業を完了したら、変更内容を分かりやすい日本語メッセージでgitコミットすること

## AI会議の役割分担

- Claude Codeは常にPlan権限で参加し、設計案の作成とコードレビューだけを行う
- Claude Codeはファイル編集、テスト実行、gitコミットを行わず、実装作業をCodexへ引き渡す
- Codexも設計議論に参加し、Claude Codeと論点を検討したうえで最終方針を決める
- Codexがファイル編集、テスト実行、gitコミットを担当する

## Git push運用ルール

- feature/* ブランチ: 作業完了時のコミット後、同名のリモートブランチへ自動pushしてよい(確認不要)
- mainへの反映やブランチ削除などそれ以外のリモート操作: 必ずユーザーの確認を得てから実行
