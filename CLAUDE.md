# CLAUDE.md

## 運用ルール

- 作業開始時に`AGENTS.md`を全文読み、このファイルと合わせて共通方針として扱う
- 両ファイルの内容が競合する場合は作業を止めてユーザーへ報告する

## CodexとClaude CodeのAI会議

- コード変更を伴う修正・実装では、CodexとClaude Codeの両方が参加するAI会議を行う
- 実装前に、Codexが現状、原因、変更案、リスク、検証方針をClaude Codeへ共有し、両者で論点を検討してから最終方針を決める
- Claude Codeは常にPlan権限で参加し、設計案の作成とコードレビューだけを行う。ファイル編集、テスト実行、gitコミット、pushは行わない
- Codexが最終判断、ファイル編集、テスト実行、gitコミット、pushを担当する
- 実装後かつコミット前に、Codexは変更差分とテスト結果をClaude Codeへ共有してレビューを受ける。指摘を反映した場合は必要なテストを再実行する
- Claude Codeを利用できない場合、CodexはAI会議やレビューを実施済みと表現せず、利用できない理由と未実施範囲をユーザーへ報告する

## Git push運用ルール

- 修正・実装依頼では、ユーザーが「コミットしない」「pushしない」「PRにする」など別の指定をしていない限り、検証済みの今回差分だけを`main`へコミットし、`origin/main`へpushする
- `main`以外で作業している場合は、差分と競合を確認してから`main`へ安全に統合する
- force push、履歴改変、無関係な変更の取り消しは行わない
- Firebase Hostingや本番ルールのデプロイは、明示的に依頼された場合だけ行う。`main`へのpushと本番デプロイは別の操作として扱う
